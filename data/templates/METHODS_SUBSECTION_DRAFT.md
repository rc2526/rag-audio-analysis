## Automated Transcript-to-Manual Analysis Pipeline

We developed a local, retrieval-augmented analysis pipeline to characterize how intervention content appeared across de-identified PMH group-session transcripts and to estimate alignment between delivered content and the intervention manual. The pipeline used de-identified transcript text generated from PMH group sessions and a session-structured version of the intervention manual. The manual was segmented into session-level and subsection-level units (for example, homework, handouts, discussion, breathing exercise, and physical activity/nutrition content) so that transcript evidence could be linked back to interpretable protocol units.

All transcript and manual content were analyzed within a local retrieval environment built from the existing `rag-audio` index. For each prespecified session-topic pair, the system ran a session-aware retrieval query of the form `Session {n} {topic_label}` to identify transcript evidence relevant to that topic. Retrieved transcript hits were expanded into small local context windows to preserve short multi-turn exchanges rather than isolated turns. To reduce reliance on lexical overlap, topic assignment for manual units and transcript-to-manual matching were implemented using embedding similarity rather than keyword scoring. Topic entries were embedded using session-labeled topic text, and manual units were embedded using their session heading, subsection heading, and content. Retrieved transcript windows were then compared against these embedding spaces to assign topic-level and manual-unit-level matches.

For each session-topic pair, we calculated an automated manual-alignment summary using three quantities: the number of expected manual units for that session-topic pair, the number of those units matched by retrieved transcript evidence, and the number of matched manual subsections. Manual-unit coverage was defined as matched manual units divided by expected manual units. Subsection coverage was defined as matched manual subsections divided by expected manual subsections. These were combined into a weighted adherence score: `0.6 * manual-unit coverage + 0.4 * subsection coverage`. Session-topic pairs were then labeled as high adherence (`>= 0.66`), moderate adherence (`>= 0.33` and `< 0.66`), or low adherence (`< 0.33`). Because these values were generated from automated retrieval and matching rather than adjudicated human coding, we treat them as computational fidelity proxies rather than definitive fidelity ratings.

To address the principal investigator’s analytic questions, we also ran four question-specific retrieval passes for each session-topic pair: facilitator reference to the topic, facilitator demonstration of the skill, participant practice in session or at home, and participant use of the skill with their child at home. For each question, the system retrieved transcript evidence windows and relevant manual units, then submitted only those retrieved materials to `gpt-oss:120b` via a remote Ollama host. The model was instructed to return structured JSON containing a short answer summary, evidence references, cited manual units, and a self-reported low/medium/high confidence label. This design constrained the model to retrieved evidence rather than the full transcript corpus and preserved traceability between each answer and the underlying transcript excerpts.

All cycle-level outputs were written to per-cycle folders and then aggregated across cycles 1-5 into summary tables for reporting. These aggregate outputs included session-topic fidelity tables, question-level answer tables, cycle-level descriptive statistics, and evidence summaries intended for downstream manuscript preparation.
